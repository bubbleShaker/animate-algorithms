# アルゴリズムカタログ（難易度別）

> CodinGame / AtCoder の Algorithm・Heuristic で頻出のものを軸に、次点で CTF・ソフトウェア開発で
> よく使うものを含めて整理する。各項目に「用途」と「可視化の勘所」を添える。
> 可視化の優先度は ★（高）〜☆（低）で示す。

## 凡例

- **用途**: どんな場面で使うか
- **可視化の勘所**: アニメーションで「何が動けば理解が進むか」
- **優先度**: mermaid + Manim 化する順番の目安

## 初級（まず動かして直感を掴む）

| アルゴリズム | 用途 | 可視化の勘所 | 優先度 |
|------------|------|------------|:---:|
| **バブルソート** | 整列の入門 | 隣接比較→交換が泡のように上がる様子 | ★★★ |
| 選択ソート | 整列の入門 | 最小値を選んで前に詰める動き | ★★ |
| 挿入ソート | 整列の入門 | 手札を正しい位置へ差し込む動き | ★★ |
| **二分探索** | ソート済み配列の高速検索 | 探索範囲が半分ずつ縮む様子 | ★★★ |
| 線形探索 | 検索の基礎 | 先頭から順に走査するポインタ | ★ |
| スタック / キュー | データ構造の基礎 | LIFO/FIFO の出入り | ★★ |
| 累積和 | 区間和の高速化 | 前計算テーブルが埋まる様子 | ★★ |

## 中級（典型問題で頻出）

| アルゴリズム | 用途 | 可視化の勘所 | 優先度 |
|------------|------|------------|:---:|
| **幅優先探索 (BFS)** | 最短経路（重みなし）・グラフ探索 | 波紋状に距離が広がる様子 | ★★★ |
| **深さ優先探索 (DFS)** | 連結成分・経路列挙・再帰 | 行けるところまで潜って戻る | ★★★ |
| クイックソート | 高速整列 | pivot 分割の再帰構造 | ★★ |
| マージソート | 安定な高速整列 | 分割統治の併合 | ★★ |
| ヒープ / 優先度付きキュー | 最大最小の高速取得 | 木の上下フィルタ | ★★ |
| しゃくとり法 | 区間問題の線形化 | 2 ポインタの伸縮 | ★★ |
| Union-Find | 連結性・グループ管理 | 木の併合・経路圧縮 | ★★★ |
| 動的計画法 (DP) | 最適化・数え上げ | DP テーブルが埋まる順序 | ★★★ |
| ダイクストラ法 | 重み付き最短経路 | 確定ノードが広がる様子 | ★★ |

## 可視化ロードマップ

```mermaid
graph LR
    A["MVP: バブルソート<br/>(パイプライン確立)"] --> B[二分探索]
    B --> C[BFS / DFS]
    C --> D[Union-Find / DP]
    D --> E[ダイクストラ等の応用]
```

1. **MVP = バブルソート** … mermaid + Manim のパイプラインをここで確立する（Issue #2）
2. 二分探索 … 「範囲が半分になる」直感を固める
3. BFS / DFS … グラフ系データフロー可視化の型を作る
4. Union-Find / DP … 状態が育つ系の可視化
5. 応用（ダイクストラ、ヒューリスティック）

> まず 1 個（バブルソート）を最後まで通し、再利用できる Manim 部品（配列バー、比較ハイライト、
> 交換アニメ）を `scenes/` の共通モジュールに切り出す方針とする。
